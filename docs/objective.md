# 主要目标
- [ ] 一个通用的整理资料的工具（输出为***文档***）
场景：课程笔记、项目申报、代码文档
产品：网站（云服务）、obsidian插件、本地app
- [ ] 一个可以根据用户需求调整工作流程的Agent
技术方案：multi-agent框架、skill...

# 次要目标
- [ ] ***先进*** 的用户交互手段（between **GUI** and **Terminal**）
- GUI：用户友好（+）、现有agent能力不足（？-）
- Terminal： 已有成熟方案（+）、**透明**（+）、需要一定技术门槛（-）
- [ ] 多种模态的资料（任意形式的输入）
- 图片（必选）
- 代码（重要）
- 视频（重要）
  **NOTE：**多模态的一致性、系统效率需要额外设计，从技术角度不一定容易解决，可能从限制用户行为上更容易实现。
- [ ] 专用/领域知识的管理
- agent能够**自主**整理经验，并根据不同的需求场景管理相应的技能、工具（管理skill）
- [ ] 可插拔的软件系统（tools）
- 管理skill的执行环境
